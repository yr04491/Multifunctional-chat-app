# 🚀 プロジェクト・コンセプト：究極のルールベース敬語変換チャット

## 1. 概要（What we build）
ベースとなるのは、時空カオス（遅延）や暗号化などのギミックを備えた多機能チャットアプリ（Next.js + Firebase）。
ここに、**「ユーザーのカジュアルな入力を、文脈に合った自然な敬語（丁寧語・尊敬語・謙譲語）に自動変換して送信する機能」**を追加する。

### 💡 最大の技術的ブレイクスルー（The Hack）
**「本番環境ではAIを一切使わない。しかし、開発フェーズでAIを限界まで酷使する」**

1. **開発時（Build time）**:
   ChatGPTなどの生成AIに大量のプロンプトを投げ、形態素解析、条件分岐、正規表現を駆使した「人間には手作業で書けない規模の、超巨大で複雑な敬語変換ルールプログラム（辞書）」を自動生成させる。
2. **本番時（Run time）**:
   本番のアプリには、その「出来上がった静的なルールベースのコード」だけを組み込んで動かす。外部のAI APIへのリクエストは一切行わない。

---

## 2. なぜこのアプローチなのか？（Why this wins hackathons）

ハッカソンにおいて、「APIを叩いて結果を返すだけの量産型AIアプリ」はもはや評価されない。
AIを単なる機能としてではなく、**「超複雑なコードを事前生成するためのコンパイラ」**として使い倒すというメタなアプローチは、「技術の使い方がスマートでハッカー気質がある」と審査員から極めて高く評価される。

さらに、プロダクトとして以下の**「4つのゼロ」**という圧倒的な優位性を持つ。

1. **💸 ランニングコスト「ゼロ」**
   外部APIを叩かないため、ユーザーが何万回メッセージを送ってもAPIの従量課金や追加のサーバー代が永久に発生しない。
2. **⚡ レスポンス時間「ゼロ」**
   AIの返答を待つ数秒のラグがない。プログラムの条件分岐と置換だけで処理されるため、ミリ秒単位で爆速変換され、チャットの命である「リアルタイム性」を一切損なわない。
3. **🔋 ユーザー端末への負担「ゼロ」**
   ブラウザで直接AIを動かす場合（WebLLMなど）に発生する「初回数GBの激重ダウンロード」や「古いスマホでのフリーズ」といった致命的なデメリットを完全に回避できる。
4. **🔒 プライバシー懸念「ゼロ」**
   ユーザーのメッセージデータは外部のAPIサーバー（OpenAIやGoogle）に一切送信されず、アプリ内で完結して変換される。

## 3. チームで挑む「最大の壁」（The Challenge）

この計画を成功させるための最大の難所は、**「AIにどんな指示（プロンプト）を与えて、どうやってその複雑な変換ルールを構築させるか」**である。

**【具体的な課題：主語の判別】**
例えば「明日行きます」という入力があった場合：
* 自分が主語なら → 「明日**参ります**」（謙譲語）
* 相手が主語なら → 「明日**いらっしゃいますか**」（尊敬語）

この文脈や主語の判別ロジック（前後のトークン分析やフラグ管理など）を、いかに「静的なルールベースのコード（正規表現やif文の羅列）」に落とし込ませるか。ここがこのプロダクト最大の腕の見せ所となる。

## 4. Next Steps
まずは「AIにルールベース辞書を作らせる」ための実験環境を構築する。

1. **アプローチの決定**: 
   - Pythonスクリプト等でAIのAPIを自動で叩き続け、少しずつ変換パターンを学習・出力させるか？
   - 手動でプロンプトを練り上げ、巨大なJSON辞書を一度に吐き出させるか？
2. **プロンプト・エンジニアリング**:
   - 敬語変換に必要な要素（丁寧語・尊敬語・謙譲語の使い分けルール）をプロンプトにどう組み込むか定義する。
3. **Next.jsへの組み込みテスト**:
   - 出力されたルールベース（仮）を `src/app/actions` 等に組み込み、実際のレスポンス速度と精度を検証する。
